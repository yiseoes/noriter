CONSTRUCTOR RULE — CRITICAL:
- Renderer constructor MUST take NO arguments: constructor() { this.canvas = null; this.ctx = null; ... }

- The initialization code calls new Renderer() then renderer.init(canvas) — this order is fixed and cannot change

init(canvas) METHOD — CRITICAL:
- MUST store: this.canvas = canvas; this.ctx = canvas.getContext('2d');
- Do NOT compute scaleX/scaleY here — canvas.offsetWidth may be 0 at load time inside an iframe
- Scale factors must be computed every frame inside render()

render(game) METHOD — CRITICAL:
- Receives the GAME OBJECT directly — access game.state, game.score etc. directly on the game object
- Do NOT call game.getGameState() — the game object's properties are accessed directly
- FIRST LINE: update scale factors every frame:
    this.scaleX = (this.canvas.offsetWidth || this.canvas.width || 800) / 800;
    this.scaleY = (this.canvas.offsetHeight || this.canvas.height || 600) / 600;
- SECOND LINE: clear the canvas completely:
    this.ctx.clearRect(0, 0, this.canvas.width, this.canvas.height);
  OR fill with background color
- Wrap all drawing in ctx.save() ... ctx.restore() — MUST be balanced (every save needs a restore)

CTX STATE HYGIENE — CRITICAL (causes invisible/broken rendering if violated):
- After changing ctx.globalAlpha, ALWAYS reset: ctx.globalAlpha = 1;
- After changing ctx.textAlign or ctx.textBaseline, reset to defaults after use
- After ctx.scale(), ctx.translate(), ctx.rotate() — wrap in ctx.save()/ctx.restore() to contain transforms
- Never leave ctx in a modified state between render calls — accumulated transforms break all future frames

COORDINATE SYSTEM:
- All game objects use 800×600 logical coordinates
- Apply scaleX/scaleY when drawing: ctx.fillRect(x * this.scaleX, y * this.scaleY, w * this.scaleX, h * this.scaleY)
- OR use ctx.save(); ctx.scale(this.scaleX, this.scaleY); /* draw in logical space */ ctx.restore();
- Font sizes should also scale: ctx.font = `${20 * Math.min(this.scaleX, this.scaleY)}px sans-serif`
