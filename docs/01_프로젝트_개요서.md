# 놀이터 (NoriTer) - 프로젝트 개요서

> **문서 버전**: v1.0
> **작성일**: 2026-03-24
> **상태**: 초안

---

## 1. 프로젝트 비전

**"요구사항 하나면, AI가 게임을 만들어드립니다."**

놀이터(NoriTer)는 AI 멀티 에이전트 기반의 미니게임 자동 개발 플랫폼이다.
사용자가 자연어로 게임 요구사항을 입력하면, 가상의 AI 게임 개발 스튜디오가
기획 → 설계 → 구현 → 테스트 → 출시까지 전 과정을 자동으로 수행한다.

---

## 2. 프로젝트 목표

| # | 목표 | 설명 |
|---|------|------|
| G-01 | 자연어 기반 게임 생성 | 사용자가 텍스트로 요구사항을 입력하면 플레이 가능한 미니게임 산출 |
| G-02 | 멀티 에이전트 협업 | CTO, 기획, 디자인, 프론트엔드, 백엔드, QA 에이전트가 부서처럼 협업 |
| G-03 | 실시간 진행 모니터링 | 각 에이전트의 작업 상태, 로그, 에러를 실시간으로 확인 가능 |
| G-04 | 감사 추적 | 전체 작업 이력, 의사결정 로그, 에러 이력을 완전히 추적 가능 |
| G-05 | 웹 기반 플랫폼 | 별도 설치 없이 브라우저에서 요구사항 입력부터 게임 플레이까지 가능 |

---

## 3. 프로젝트 범위

### 3.1 포함 (In Scope)

- 웹 기반 미니게임 생성 (HTML5 / Canvas / Phaser.js)
- 멀티 에이전트 파이프라인 (6개 부서)
- 실시간 로그 스트리밍 (진행상황, 에러, 감사 로그)
- 프로젝트(요청) 관리 — 생성, 목록, 상세, 재시도, 삭제, 수정 요청
- 생성된 게임 미리보기 및 다운로드
- 에이전트 간 커뮤니케이션 로그 열람

### 3.2 제외 (Out of Scope)

- 모바일 네이티브 앱
- 3D 게임 (Unity, Unreal 등)
- 멀티플레이어/서버 기반 게임
- 사용자 인증/결제 시스템 (v1 기준)
- 게임 에셋(이미지/사운드) 직접 생성 (v1에서는 기본 도형·CSS 기반)

## 4. 핵심 컨셉: AI 게임 스튜디오

놀이터는 하나의 **가상 게임 개발 회사**처럼 동작한다.

```
사용자(클라이언트)
    │
    │  "뱀파이어 서바이벌 류 미니게임 만들어줘"
    ▼
┌──────────────────────────────────────────┐
│          🏢 놀이터 AI 스튜디오             │
│                                          │
│  [CTO]       전체 기술 아키텍처 결정        │
│  [기획팀]     게임 기획서 작성              │
│  [디자인팀]   UI/UX 설계, 스타일 가이드     │
│  [프론트팀]   게임 클라이언트 구현           │
│  [백엔드팀]   게임 로직·데이터 구현          │
│  [QA팀]      테스트, 디버깅, 품질 보증      │
│                                          │
│  ──── 파이프라인 ────                     │
│  기획 → CTO 검토 → 설계 → 구현            │
│  → QA 테스트 → 디버깅 → 완성              │
└──────────────────────────────────────────┘
    │
    ▼
🎮 플레이 가능한 미니게임
```

## 5. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| **백엔드** | Java 17, Spring Boot 3.x, Maven | 메인 서버 |
| **프론트엔드** | React 18, TypeScript, Vite | 관리 UI |
| **AI 엔진** | Claude API (Anthropic SDK for Java) | 에이전트 두뇌 |
| **실시간 통신** | SSE (Server-Sent Events) | 진행상황 스트리밍 |
| **데이터베이스** | PostgreSQL | 프로젝트·로그·감사 |
| **게임 출력** | HTML5, Canvas, Phaser.js | 생성되는 미니게임 |
| **빌드 도구** | Maven (백엔드), Vite (프론트) | |

## 6. 사용자 시나리오 (핵심 흐름)

### 시나리오 1: 게임 생성 요청

1. 사용자가 웹 UI에서 게임 요구사항을 자연어로 입력
2. 시스템이 요청을 접수하고 프로젝트를 생성
3. 기획팀 에이전트가 요구사항을 분석하여 게임 기획서 작성
4. CTO 에이전트가 기획서를 검토하고 기술 아키텍처 결정
5. 디자인팀 에이전트가 UI/UX 설계
6. 프론트엔드/백엔드 에이전트가 게임 코드 구현
7. QA 에이전트가 코드 검증 및 테스트
8. 문제 발견 시 해당 에이전트에게 수정 요청 (반복)
9. 완성된 게임을 사용자에게 제공 (미리보기 + 다운로드)

### 시나리오 2: 진행 모니터링

1. 사용자가 대시보드에서 현재 프로젝트 상태 확인
2. 각 에이전트(부서)별 작업 진행률 실시간 확인
3. 에이전트 간 대화 로그 열람
4. 에러 발생 시 알림 및 에러 상세 확인

### 시나리오 3: 결과물 확인

1. 게임 완성 알림 수신
2. 웹 UI에서 바로 게임 미리보기 (iframe)
3. 소스 코드 확인 및 다운로드
4. 필요 시 수정 요청 → 재작업 파이프라인 진행

## 7. 주요 용어 정의

| 용어 | 정의 |
|------|------|
| **프로젝트** | 사용자의 게임 생성 요청 1건 단위 |
| **에이전트** | 특정 역할을 수행하는 AI 단위 (= 부서) |
| **파이프라인** | 에이전트들이 순차/병렬로 작업하는 전체 흐름 |
| **스테이지** | 파이프라인 내 하나의 작업 단계 (기획, 설계, 구현 등) |
| **아티팩트** | 에이전트가 생산한 산출물 (기획서, 코드, 테스트 결과 등) |
| **감사 로그** | 에이전트의 모든 의사결정 및 작업 이력 |
