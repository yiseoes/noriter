> **문서 버전**: v1.0
> **작성일**: 2026-03-24
> **상태**: 초안

---

## 1. 기능 요구사항 ID 체계

```
NT-[카테고리]-[번호]

카테고리:
  PRJ  = 프로젝트 관리
  AGT  = 에이전트·파이프라인
  MON  = 모니터링·로그
  GAM  = 게임 출력·미리보기
  SYS  = 시스템·공통
```

### NT-PRJ-001: 게임 생성 요청

| 항목 | 내용 |
|------|------|
| **우선순위** | 🔴 필수 |
| **설명** | 사용자가 자연어로 미니게임 요구사항을 입력하여 게임 생성을 요청한다 |
| **선행 조건** | Claude API 키가 설정되어 있어야 함 |
| **주요 흐름** | 1. 사용자가 요구사항 텍스트 입력<br>2. (선택) 게임 장르/스타일 태그 선택<br>3. "생성 요청" 버튼 클릭<br>4. 시스템이 프로젝트 생성 후 파이프라인 자동 시작<br>5. 대시보드로 이동하여 진행상황 확인 |
| **입력** | - 요구사항 텍스트 (필수, 최소 10자)<br>- 게임 장르 태그 (선택): 퍼즐, 액션, 아케이드, 슈팅, 전략 등<br>- 프로젝트명 (선택, 미입력 시 자동 생성) |
| **출력** | 프로젝트 ID, 프로젝트 상태(CREATED → IN_PROGRESS) |
| **예외** | - 요구사항 10자 미만: 에러 메시지 표시<br>- API 키 미설정: 설정 페이지 안내<br>- 동시 실행 제한 초과: 대기열 안내 |

### NT-AGT-001: 파이프라인 실행

| 항목 | 내용 |
|------|------|
| **우선순위** | 🔴 필수 |
| **설명** | 에이전트들이 순서대로 작업을 수행하는 파이프라인을 실행한다 |
| **파이프라인 스테이지** | |

```
STAGE 1: 기획 (Planning)
  ├─ 담당: 기획팀 에이전트
  ├─ 입력: 사용자 요구사항 원문
  ├─ 산출물: 게임 기획서 (장르, 규칙, 조작법, 승리조건 등)
  └─ 완료 조건: 기획서 JSON 생성

STAGE 2: CTO 검토 (Architecture Review)
  ├─ 담당: CTO 에이전트
  ├─ 입력: 게임 기획서
  ├─ 산출물: 기술 아키텍처 문서 (파일 구조, 기술 선택, 모듈 분리)
  └─ 완료 조건: 아키텍처 문서 JSON 생성

STAGE 3: 설계 (Design)
  ├─ 담당: 디자인팀 에이전트
  ├─ 입력: 기획서 + 아키텍처 문서
  ├─ 산출물: UI/UX 설계서 (색상, 레이아웃, CSS 스타일 가이드)
  └─ 완료 조건: 디자인 스펙 JSON 생성

STAGE 4: 구현 (Implementation)
  ├─ 담당: 프론트엔드 + 백엔드 에이전트 (병렬)
  ├─ 입력: 기획서 + 아키텍처 + 디자인 스펙
  ├─ 산출물: 게임 소스 코드 (HTML, CSS, JS)
  └─ 완료 조건: 실행 가능한 코드 파일 생성

STAGE 5: QA 테스트 (Quality Assurance)
  ├─ 담당: QA팀 에이전트
  ├─ 입력: 게임 소스 코드 + 기획서
  ├─ 산출물: 테스트 결과 보고서 (버그 목록, 개선점)
  └─ 완료 조건: 테스트 통과 또는 버그 리포트 생성

STAGE 6: 디버깅 (Debug) — 조건부
  ├─ 담당: 해당 에이전트 (버그 유형에 따라 CTO가 배정)
  ├─ 입력: 버그 리포트 + 소스 코드
  ├─ 산출물: 수정된 소스 코드
  └─ 완료 조건: QA 재테스트 통과 (최대 3회 반복)

STAGE 7: 출시 (Release)
  ├─ 담당: 시스템
  ├─ 처리: 최종 코드 패키징, 미리보기 URL 생성
  └─ 완료 조건: 게임 접근 가능
```

| **동시 실행** | STAGE 4에서 프론트/백엔드 에이전트 병렬 실행 |
| **최대 반복** | QA ↔ 디버깅 순환 최대 3회 |
| **타임아웃** | 스테이지당 최대 5분, 전체 파이프라인 최대 30분 |

| 항목 | 내용 |
|------|------|
| **우선순위** | 🔴 필수 |
| **설명** | 각 에이전트의 현재 상태를 관리한다 |
| **상태 종류** | |


### NT-MON-001: 실시간 로그 스트리밍

| 항목 | 내용 |
|------|------|
| **우선순위** | 🔴 필수 |
| **설명** | 파이프라인 진행 중 에이전트의 작업 로그를 실시간으로 스트리밍한다 |
| **구현 방식** | SSE (Server-Sent Events) |
| **로그 종류** | |

```
[INFO]    일반 진행 로그      "기획팀이 게임 기획서 작성을 시작합니다"
[AGENT]   에이전트 대화 로그    "CTO → 프론트팀: Canvas 기반으로 구현해주세요"
[WARN]    경고 로그           "API 응답 지연 감지 (3.2초)"
[ERROR]   에러 로그           "Claude API 호출 실패: Rate Limit"
[AUDIT]   감사 로그           "프로젝트 P-001 생성됨, 요청자: user"
[DEBUG]   디버그 로그          "프롬프트 토큰 수: 2,340"
```

| **표시 방식** | - 로그 레벨별 색상 구분<br>- 에이전트별 필터링<br>- 자동 스크롤 (최신 로그 하단) |
