## 2. 계층 구조 (Layered Architecture)

| 계층 | 역할 | 주요 컴포넌트 |
|------|------|---------------|
| **Presentation** | REST API 엔드포인트, SSE 스트리밍 | Controller, SSE Emitter |
| **Service** | 비즈니스 로직, 트랜잭션 관리 | Service 클래스 |
| **Agent Engine** | 에이전트 실행, 파이프라인 오케스트레이션 | Orchestrator, Agent, MessageBus |
| **Infrastructure** | 외부 시스템 연동, 데이터 접근 | Repository, Claude API Client, FileStorage |

## 3. 에이전트 엔진 상세 설계

### 3.1 에이전트 구조

모든 에이전트는 공통 인터페이스 `BaseAgent`를 구현한다.

```
<<interface>>
BaseAgent
├── execute(context: AgentContext): AgentResult
├── getRole(): AgentRole
├── getStatus(): AgentStatus
└── cancel(): void

구현체:
├── CtoAgent          — 아키텍처 결정, 디버깅 배정
├── PlanningAgent     — 게임 기획서 작성
├── DesignAgent       — UI/UX 설계
├── FrontendAgent     — 게임 클라이언트 코드 생성
├── BackendAgent      — 게임 로직 코드 생성
└── QaAgent           — 코드 검증, 버그 리포트
```

### 3.2 AgentContext (에이전트 실행 컨텍스트)

```
AgentContext
├── projectId         — 프로젝트 ID
├── stageId           — 현재 스테이지 ID
├── requirement       — 사용자 원본 요구사항
├── previousArtifacts — 이전 스테이지 산출물 목록
├── messages          — 수신된 메시지 목록
└── config            — 에이전트 설정 (프롬프트 템플릿 등)
```

### 3.3 AgentResult (에이전트 실행 결과)

```
AgentResult
├── status            — SUCCESS / FAILED / NEEDS_REVIEW
├── artifacts         — 생성된 산출물 목록
├── messages          — 다음 에이전트에게 보낼 메시지
├── logs              — 작업 로그 목록
└── errorInfo         — 실패 시 에러 정보 (nullable)
```

### 3.4 PipelineOrchestrator (파이프라인 오케스트레이터)

파이프라인의 전체 흐름을 제어하는 핵심 컴포넌트.

```
PipelineOrchestrator
├── startPipeline(projectId): void
│     → STAGE 순서대로 에이전트 실행
│     → 각 스테이지 완료 시 다음 스테이지로 전이
│
├── handleStageComplete(stageId, result): void
│     → 산출물 저장
│     → 다음 스테이지 결정 및 실행
│     → QA 실패 시 디버깅 스테이지로 분기
│
├── handleStageFailure(stageId, error): void
│     → 재시도 정책 확인 (NT-AGT-005)
│     → 재시도 초과 시 파이프라인 FAILED 처리
│
├── cancelPipeline(projectId): void
│     → 현재 에이전트에 cancel() 호출
│     → 프로젝트 상태 CANCELLED 전환
│
└── retryPipeline(projectId, fromStage?): void
      → 처음부터 또는 특정 스테이지부터 재실행
```

### 3.5 MessageBus (에이전트 간 통신)

```
MessageBus
├── send(message: AgentMessage): void
│     → 메시지 DB 저장 (감사 로그)
│     → SSE로 실시간 클라이언트 전달
│
├── getMessages(projectId, agentRole?): List<AgentMessage>
│     → 프로젝트별·에이전트별 메시지 조회
│
└── AgentMessage
      ├── id
      ├── projectId
      ├── fromAgent      — 발신 에이전트 (AgentRole)
      ├── toAgent        — 수신 에이전트 (AgentRole)
      ├── type           — HANDOFF, REVIEW_REQUEST, BUG_REPORT, FEEDBACK
      ├── content        — 메시지 본문
      ├── artifactRef    — 산출물 참조 ID (nullable)
      └── createdAt
```

## 4. 파이프라인 상태 전이도

### 4.1 프로젝트 상태

```
CREATED ──→ IN_PROGRESS ──→ COMPLETED
                │                 │
                │                 └──→ REVISION (수정요청 시)
                │                        │
                │                        └──→ IN_PROGRESS (재작업)
                │
                ├──→ FAILED
                │      │
                │      ├──→ IN_PROGRESS (재시도)
                │      └──→ (삭제)
                │
                └──→ CANCELLED
                       └──→ (삭제)
```

| 상태 | 설명 | 전이 가능 상태 |
|------|------|----------------|
| CREATED | 프로젝트 생성됨, 파이프라인 시작 전 | IN_PROGRESS |
| IN_PROGRESS | 파이프라인 실행 중 | COMPLETED, FAILED, CANCELLED |
| COMPLETED | 게임 생성 완료 | REVISION |
| REVISION | 수정 요청 접수, 재작업 중 | IN_PROGRESS |
| FAILED | 파이프라인 실패 | IN_PROGRESS (재시도) |
| CANCELLED | 사용자에 의해 중단됨 | - |

### 4.2 스테이지 상태

```
PENDING ──→ IN_PROGRESS ──→ COMPLETED ──→ (다음 스테이지)
                │
                ├──→ FAILED ──→ RETRYING ──→ IN_PROGRESS
                │                   │
                │                   └──→ FAILED (최종)
                │
                └──→ SKIPPED (수정요청 시 불필요한 스테이지)
```

### 4.3 파이프라인 흐름 상세

```
[STAGE 1: 기획]
     │
     ▼
[STAGE 2: CTO 검토]
     │
     ▼
[STAGE 3: 디자인]
     │
     ▼
[STAGE 4: 구현] ──── 프론트엔드 ────┐
     │                              │ (병렬)
     │              백엔드 ──────────┘
     │                    │
     │              합류 (merge)
     ▼
[STAGE 5: QA 테스트]
     │
     ├── 통과 ──→ [STAGE 7: 출시] ──→ COMPLETED
     │
     └── 실패 ──→ [STAGE 6: 디버깅]
                       │
                       └──→ [STAGE 5: QA 재테스트]
                              │
                              ├── 통과 ──→ [STAGE 7: 출시]
                              └── 실패 ──→ (최대 3회 반복 후 FAILED)
```

## 5. 기술 컴포넌트 상세

### 5.1 Claude API 연동 (ClaudeApiClient)

```
ClaudeApiClient
├── sendPrompt(prompt, systemPrompt): String
│     → Claude API 호출, 응답 텍스트 반환
│     → 재시도 정책 내장 (NT-AGT-005: 3회, 지수 백오프)
│
├── sendPromptStreaming(prompt, systemPrompt, listener): void
│     → 스트리밍 응답, 토큰 단위로 listener에 전달
│     → 실시간 로그 스트리밍에 활용
│
├── 설정
│     ├── model: claude-sonnet-4-6 (기본, 에이전트별 변경 가능)
│     ├── maxTokens: 4096 (기본)
│     └── temperature: 에이전트별 상이
│
└── 토큰 사용량 추적
      → 프로젝트별·에이전트별 토큰 사용량 기록
```

### 5.2 SSE 실시간 스트리밍 (SSE Emitter)

```
SSE 이벤트 타입:
├── log          — 실시간 로그 (NT-MON-001)
├── stage-update — 스테이지 상태 변경 (NT-MON-002)
├── agent-msg    — 에이전트 간 메시지 (NT-MON-003)
├── error        — 에러 발생 (NT-MON-004)
├── complete     — 파이프라인 완료
└── cancelled    — 파이프라인 취소

엔드포인트: GET /api/projects/{id}/stream
```

### 5.3 파일 스토리지 (FileStorage)

```
FileStorage
├── 저장 경로 구조:
│     workspace/
│     └── {projectId}/
│           ├── artifacts/
│           │     ├── plan.json          — 기획서 (STAGE 1)
│           │     ├── architecture.json  — 아키텍처 (STAGE 2)
│           │     ├── design.json        — 디자인 스펙 (STAGE 3)
│           │     └── test-report.json   — QA 리포트 (STAGE 5)
│           │
│           └── game/
│                 ├── index.html
│                 ├── style.css
│                 ├── game.js
│                 └── (기타 게임 파일)
│
├── saveArtifact(projectId, type, content): String (파일 경로)
├── getArtifact(projectId, type): String (내용)
├── saveGameFiles(projectId, files): void
├── getGameFiles(projectId): List<GameFile>
└── packageAsZip(projectId): byte[] (NT-GAM-003)
```

## 9. 정합성 검증 매트릭스

| 기능 요구사항 (02) | 아키텍처 컴포넌트 | 담당 계층 |
|--------------------|-------------------|-----------|
| NT-PRJ-001 게임 생성 요청 | ProjectController → ProjectService → PipelineOrchestrator | Presentation → Service → Agent |
| NT-PRJ-002 목록 조회 | ProjectController → ProjectService | Presentation → Service |
| NT-PRJ-003 상세 조회 | ProjectController → ProjectService + ArtifactService | Presentation → Service |
| NT-PRJ-004 재시도 | ProjectController → PipelineOrchestrator.retryPipeline() | Presentation → Agent |
| NT-PRJ-005 삭제 | ProjectController → ProjectService | Presentation → Service |
| NT-PRJ-006 수정 요청 | ProjectController → PipelineOrchestrator (부분 재실행) | Presentation → Agent |
| NT-AGT-001 파이프라인 실행 | PipelineOrchestrator → 각 Agent | Agent Engine |
| NT-AGT-002 메시지 전달 | MessageBus → AgentMessageRepository | Agent → Infrastructure |
