> **문서 버전**: v1.0
> **작성일**: 2026-03-24
> **상태**: 초안

## 1. 화면 설계 원칙

| 항목 | 원칙 |
|------|------|
| **프레임워크** | React 18 + TypeScript + Vite (01_개요서 §5, 03_아키텍처 §7) |
| **라우팅** | React Router v6 (SPA) |
| **상태 관리** | React Query (서버 상태) + useState/useReducer (로컬 상태) |
| **SSE 연동** | EventSource API (API-SSE-001, NT-MON-001) |
| **스타일링** | Tailwind CSS + shadcn/ui (일관된 UI 컴포넌트) |
| **반응형** | 데스크톱 우선 (NFR-04: Chrome, Edge 최신 2개 버전) |
| **배포** | Spring Boot 정적 리소스로 서빙 (03_아키텍처 §8) |

## 2. 화면 ID 체계

```
SCR-[카테고리]-[번호]

카테고리:
  PRJ  = 프로젝트 관련 화면
  MON  = 모니터링·로그
  GAM  = 게임 출력
  SYS  = 시스템·설정
  COM  = 공통 컴포넌트
```

## 4. 전체 라우팅 구조

```
/                         → SCR-PRJ-001  게임 생성 요청 (홈)

/projects/:id             → SCR-PRJ-003  프로젝트 상세
/projects/:id?tab=log     → SCR-MON-001  실시간 로그 (탭)

/audit-logs               → SCR-MON-003  감사 로그
/settings                 → SCR-SYS-001  설정
```

### 6.1 SCR-PRJ-001: 게임 생성 요청 (홈)

> 관련: NT-PRJ-001, API-PRJ-001

사용자가 자연어로 게임 요구사항을 입력하는 **서비스의 첫 화면**.

#### 와이어프레임

```
┌──────────────────────────────────────────────────────┐
│  헤더                                                 │
├──────────────────────────────────────────────────────┤
│                                                      │
│              🎮 놀이터 NoriTer                         │
│         "요구사항 하나면, AI가 게임을 만들어드립니다."      │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │  프로젝트명 (선택)                              │    │
│  │  ┌──────────────────────────────────────┐    │    │
│  │  │ 뱀파이어 서바이벌                        │    │    │
│  │  └──────────────────────────────────────┘    │    │
│  │                                              │    │
│  │  게임 장르 (선택)                              │    │
│  │  [퍼즐] [액션] [아케이드] [슈팅] [전략] [기타]   │    │
│  │                                              │    │
│  │  요구사항 *                                    │    │
│  │  ┌──────────────────────────────────────┐    │    │
│  │  │ 뱀파이어 서바이벌 류 미니게임을 만들어줘.  │    │    │
│  │  │ 사방에서 몬스터가 몰려오고 자동 공격으로   │    │    │
│  │  │ 처치하는 게임...                        │    │    │
│  │  │                                      │    │    │
│  │  │                                      │    │    │
│  │  └──────────────────────────────────────┘    │    │
│  │  최소 10자 이상 입력해주세요        12/10 ✓    │    │
│  │                                              │    │
│  │          [ 🎮 게임 생성 요청 ]                 │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
│  ── 최근 프로젝트 ──                                   │
│  ┌────────┐ ┌────────┐ ┌────────┐                    │
│  │테트리스  │ │슈팅게임 │ │퍼즐게임 │                    │
│  │✅ 완료  │ │🔄 진행중│ │❌ 실패  │                    │
│  └────────┘ └────────┘ └────────┘                    │
│                                                      │
└──────────────────────────────────────────────────────┘
```

#### 구성 요소

| # | 요소 | 타입 | 필수 | 설명 | 관련 |
|---|------|------|------|------|------|
| 1 | 프로젝트명 | 텍스트 입력 | X | 최대 100자, 미입력 시 자동 생성 | NT-ERR-V002 |
| 2 | 게임 장르 | 태그 선택 (단일) | X | PUZZLE, ACTION, ARCADE, SHOOTING, STRATEGY, OTHER | API-PRJ-001 genre |
| 3 | 요구사항 | 텍스트 영역 | O | 최소 10자, 실시간 글자 수 표시 | NT-ERR-V001 |
| 4 | 생성 버튼 | 버튼 | - | 요구사항 10자 미만 시 비활성화 | API-PRJ-001 |
| 5 | 최근 프로젝트 | 카드 목록 | - | 최근 3~5개, 클릭 시 상세 이동 | API-PRJ-002 |

#### 사용자 흐름

```
1. 홈 페이지 진입
2. (선택) 프로젝트명 입력
3. (선택) 장르 태그 선택
4. 요구사항 텍스트 입력 (10자 이상)
5. "게임 생성 요청" 버튼 클릭
6. API-PRJ-001 호출
   ├─ 성공 → /projects/:id 로 이동 (자동)
   ├─ 400 (NT-ERR-V001~V003) → 필드 아래 에러 메시지 표시
   └─ 503 (NT-ERR-S001) → "API 키를 먼저 설정해주세요" 토스트 + 설정 페이지 링크
```

#### 유효성 검증 (프론트엔드)

| 필드 | 규칙 | 에러 메시지 | 에러 코드 |
|------|------|-----------|-----------|
| requirement | 최소 10자 | "요구사항은 최소 10자 이상 입력해주세요" | NT-ERR-V001 |
| name | 최대 100자 | "프로젝트명은 100자를 초과할 수 없습니다" | NT-ERR-V002 |
| genre | enum 값 | "유효하지 않은 장르입니다" | NT-ERR-V003 |

#### SSE 실시간 연동 (IN_PROGRESS / REVISION 상태일 때)

```
EventSource 연결: GET /api/projects/{id}/stream (API-SSE-001)

이벤트 핸들링:
├─ stage-update → 파이프라인 타임라인 갱신, 진행률 갱신
├─ log          → 로그 탭에 실시간 추가
├─ agent-msg    → 대화 탭에 실시간 추가

```

### 10.1 연결 관리

```
연결 시점: 프로젝트 상세 페이지 진입 시 (status = IN_PROGRESS 또는 REVISION)

재연결: 연결 끊김 시 3초 후 자동 재연결 (최대 5회)
```

### 10.2 이벤트별 UI 반영

| SSE 이벤트 | 반영 대상 | 동작 |
|-----------|----------|------|
| `log` | SCR-MON-001 (로그 탭) | LogList에 항목 추가, 자동 스크롤 |
| `stage-update` | PipelineTimeline | 해당 스테이지 상태 아이콘 변경, 진행률 갱신 |
| `agent-msg` | SCR-MON-002 (대화 탭) | MessageList에 버블 추가 |
| `error` | Toast + SCR-MON-001 | 에러 토스트 표시 + 로그에 추가 |
| `complete` | ProjectHeader + 탭 | 상태 → COMPLETED, 미리보기 탭 활성화, 완성 토스트 |
| `cancelled` | ProjectHeader | 상태 → CANCELLED, SSE 연결 해제 |
