# 놀이터 (NoriTer) - 프로젝트 디렉토리 구조서

> **문서 버전**: v1.0
> **작성일**: 2026-03-24
> **상태**: 초안

---

## 1. 설계 원칙

| 항목 | 원칙 |
|------|------|
| **모노레포** | 백엔드(Spring Boot) + 프론트엔드(React) 단일 저장소 |
| **백엔드 패키지** | 계층형 (03_아키텍처 §2 Layered Architecture) |
| **프론트엔드 구조** | 기능 기반 (Feature-based) — 화면 단위로 디렉토리 분리 |
| **네이밍 규칙** | 백엔드: PascalCase(클래스), camelCase(변수) / 프론트: PascalCase(컴포넌트), camelCase(함수·변수) |
| **배포** | React 빌드 → Spring Boot 정적 리소스로 서빙 (03_아키텍처 §8) |

## 7. 빌드 및 배포 구조

> 참조: 03_아키텍처 §8 배포 구조

### 7.1 빌드 프로세스

```
1. 프론트엔드 빌드
   $ cd frontend
   $ npm install
   $ npm run build           → frontend/dist/ 생성

2. 빌드 결과물 복사
   frontend/dist/* → backend/src/main/resources/static/

3. 백엔드 빌드
   $ cd backend
   $ mvn clean package       → backend/target/noriter-0.0.1.jar

4. 실행
   $ java -jar noriter-0.0.1.jar
   → http://localhost:8080 (Spring Boot + React SPA)
```

### 7.2 개발 모드 (프론트/백 분리 실행)

```
백엔드:
  $ cd backend
  $ mvn spring-boot:run -Dspring-boot.run.profiles=dev
  → http://localhost:8080 (API)

프론트엔드:
  $ cd frontend
  $ npm run dev
  → http://localhost:5173 (Vite dev server, proxy → :8080)
```

### 7.3 vite.config.ts (프록시 설정)

```typescript
// frontend/vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': 'http://localhost:8080'    // API 프록시
    }
  }
})
```
