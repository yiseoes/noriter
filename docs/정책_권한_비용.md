# 놀이터(NoriTer) — 권한·비용 정책 (2026-04-08)

## 1. 역할 정의

| 역할 | 조건 | 설명 |
|------|------|------|
| **게스트** | 비로그인 (X-Guest-Id) | 체험용, localStorage UUID 기반 식별 |
| **USER** | 이메일 회원가입 | 일반 로그인 유저 |
| **ADMIN** | DB 직접 설정 (이서 계정) | 모든 기능 무제한 |
| **TEST_ADMIN** | (최종 단계 구현 예정) | 어드민 화면 구경만 가능 |

## 2. 게임 생성 정책

| 역할 | 생성 횟수 | API 호출 | 비고 |
|------|----------|---------|------|
| 게스트 | **1회** | 실제 Claude API | guestId당 1개 제한 |
| USER | **3회** | 실제 Claude API | userId당 3개 제한 |
| ADMIN | **무제한** | 실제 Claude API | 제한 없음 |

## 3. 수정 요청(피드백) 정책

| 역할 | 수정 횟수 | 조건 |
|------|----------|------|
| 게스트 | **0회** | 수정 불가 (1회 생성만) |
| USER | **프로젝트당 2회** | COMPLETED 상태에서만 요청 가능 |
| ADMIN | **무제한** | 제한 없음 |

## 4. 템플릿 미리보기 (데모)

- 비용: **0원** (Claude API 미사용)
- 목적: "이런 게임이 만들어져요" 쇼케이스
- 구성: 뱀파이어 서바이벌 / 테트리스 / 슈팅 3종
- 동작: 파이프라인 애니메이션 재생 → 미리 만들어둔 템플릿 게임 연결
- 누구나 이용 가능 (게스트/유저/어드민)

## 5. 삭제·중단·재시도 정책

| 동작 | 가능 상태 | 소유권 |
|------|----------|--------|
| 삭제 | CREATED, COMPLETED, FAILED, CANCELLED | 본인만 |
| 중단 | IN_PROGRESS, REVISION | 본인만 |
| 재시도 | FAILED | 본인만 |

## 6. 소유권 검증 (isOwnedBy)

```
① project.userId 있고 + 요청 userId 있음 → userId 비교
② project.userId null + project.guestId 있음 → guestId 비교
③ 둘 다 null (레거시) → 접근 허용
④ 그 외 → 403 PROJECT_ACCESS_DENIED
```

## 7. 인증 흐름

- **회원가입**: 이메일 형식 + 비밀번호 6자 이상 → JWT 토큰 발급
- **로그인**: 자유 형식 ID (이서 계정 '1' 대응) → JWT 토큰 발급
- **요청 헤더**: `Authorization: Bearer {JWT}` + `X-Guest-Id: {guestId}`
- 로그인 유저는 userId 기반, 비로그인은 guestId 기반

## 8. 비용 예상 (Claude Sonnet 기준)

| 항목 | 토큰 예상 | 비용 예상 |
|------|----------|----------|
| 게임 1회 생성 (6단계) | ~20K 토큰 | ~$0.10 |
| 수정 1회 (3단계: CTO→수정→QA) | ~10K 토큰 | ~$0.05 |
| 디버그 1회 (자동) | ~10K 토큰 | ~$0.05 |

최악의 시나리오 (지인 10명 전부 이용):
- 생성: 10명 × 3회 = 30회 → ~$3.00
- 수정: 30프로젝트 × 2회 = 60회 → ~$3.00
- **총 예상: ~$6.00**
